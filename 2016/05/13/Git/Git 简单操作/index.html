Git 简单操作

  • git

前言

　　Git 是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。假设你已经对 git 有一定的认识，大概知道 git 的工作原理。通常，熟练使用下面的命令基本上就可以满足工作中的需求了。毕竟是使用工具而不是研究工具，所以只介绍了 git 的常用命令。

clone

将 git 库克隆到本地。

pull

　　将当前分支关联的远端分支合并到本地分支。pull 操作可带参数，合并指定分支。git pull 实际上是一个组合操作，由 git fetch，git merge 合并而成。带参的 pull 操作比较复杂这里就不介绍了。

branch

查看所有分支。

checkout

　　切换并创建分支，new_branch 为新分支的名字，start_point 为新分支的 HEAD。如果 start_point 为空，则以当前分支作为 HEAD。如果 start_point 为 remotes/origin/feature_xxx，则以它作为 HEAD，并和本地分支关联。

status

查看当前 git 库的状态。

diff

查看当前文件的修改。指定 file 可以只查看某个文件的修改。

add

将文件加入到 git 库中。如果 new_file 为文件夹，则会将文件夹中所有文件加入。

commit

将修改提交到本地，并填写修改日志。

push

　　将当前分支合并到关联的远端分支。如果当前分支没有关联的远端分支，git 会提示你指定远端分支。第二个命令可以将本地分支推送到远端并进行关联，下次就只需要 git push 就可以了。 注意，这里故意使用 cur_branch 是希望保持本地分支和远端分支名字的一致。 Keep It Simple, Stupid. :)

log

查看当前分支上的修改日志。指定 file 能够查看某个文件的修改日志。
