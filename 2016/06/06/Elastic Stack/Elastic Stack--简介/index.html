Elastic Stack

Elastic Stack

　　Elastic Stack 是一个实时分布式数据统计分析的技术栈。通常由 Beats、Logstash、Elastic Search、Kibana 组成的。Elastic Search 作为 Elastic Stack 的核心是一个易扩展的分布式实时搜索引擎，与同样是基于 Apache Lucene 框架的 Solr 相比，Elastic Search 对不断更新的数据有着更高的查询效率。在实时日志处理领域 Elastic Stack 是开源项目中最好的选择之一。对于日志处理的场景 Elastic Stack 的组成如下图:


Filebeat

　　Filebeat 用于监控文件的变化，将新增的日志输出到 Logstash 或者直接输出到 Elastic Search。当然，Filebeat 也支持输出 Redis、Kafka 等等。在上图的结构中，Filebeat 将内容输出到 Redis 中。这样做是为了防止后端的消费者处理过慢，使用 Redis 作为缓冲队列。同时也可以隔离 Filebeat 与 Logstash 之间的故障。

Logstash

　　Logstash 用于数据的收集、解析、过滤和传输。通常我们使用 Logstash 来过滤错误的数据、结构化数据然后将数据输出到 Elastic Search。当然 Logstash 也支持输出到 file、hdfs 等等。Logstash 通常不会直接接收来自 Filebeat 的数据，因为 Logstash 处理速度会比 Filebeat 的慢，需要一个队列进行保护。

Kibana

　　Kibana 为 Elastic Search 提供数据可视化功能。通过 Kibana 搜索、查看、聚合存放在 Elastic Search 中的数据，使用 Kibana 提供的各种仪表盘能够非常方便的进行数据分析与可视化。
